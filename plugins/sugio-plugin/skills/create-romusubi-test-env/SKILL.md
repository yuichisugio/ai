# romusubi テスト環境作成

romusubi 向けのテスト環境作成はリポジトリごとに差が出やすいため、まず既存の自動化手順を確認してから実行する。

## 引数

- 第1引数: テストブランチ名に付けるサフィックス。未指定なら現在のブランチ名から安全なサフィックスを作る。

## 手順

1. 現在のリポジトリとブランチを確認する。
2. `package.json`、`Makefile`、`scripts/`、README、CI 設定などを探索して、既存のテスト環境デプロイ手順を特定する。
3. テスト用ブランチ名を `test/YYYY-MM-DD-{suffix}` 形式で決める。
4. push やデプロイの前に、対象リポジトリ・現在ブランチ・作成予定ブランチ名をユーザーに明示して確認を取る。
5. 確認後に既存の手順へ沿って test 環境を作成または更新する。

## 守ること

- 既存の deploy 用スクリプトや運用ルールが見つからない場合は、推測で push しない。
- 複数リポジトリを開いている可能性があるため、実行前に対象リポジトリを必ず再確認する。
- デプロイ後は、作成したブランチ名と確認先 URL を簡潔にまとめて共有する。
