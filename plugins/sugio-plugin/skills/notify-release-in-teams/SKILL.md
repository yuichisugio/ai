## 概要

カレントブランチと `production` ブランチの差分を調査し、社内利用ユーザー向けのTeamsリリース通知文をプレーンテキストで出力するスキル。

## 手順

### 1. スクリプトで差分情報を取得する

`skills/notify-release-in-teams/scripts/get_diff.sh` を実行して、productionブランチとの差分を取得する。

```bash
bash plugins/sugio-plugin/skills/notify-release-in-teams/scripts/get_diff.sh
```

### 2. 変更内容を社内ユーザー向けに要約する

スクリプトの出力をもとに、変更内容を**社内の一般利用ユーザー**が理解できる言葉で短文にまとめる。

- 技術用語・実装の詳細は使わない
- 「何が変わったか」「どう便利になったか」の観点で書く
- 1項目につき1〜2文の短文にする
- 例（悪い）: 「APIのレスポンスキャッシュ機構をRedisに移行」
- 例（良い）: 「画面の読み込み速度が改善されました」

### 3. テンプレートを埋めて出力する

以下のテンプレートに沿い、開発内容の箇条書きを埋めて出力する。

テンプレート構造：

```
表題の件、リリースいたします。

リリースの内容
1. （変更点1）
2. （変更点2）

よろしくお願いいたします🙇‍♀️
```

### 4. 出力

マークダウン記法は一切使わず、プレーンテキストとしてそのまま出力する。
コードブロックや `#` などの記号は不要。
