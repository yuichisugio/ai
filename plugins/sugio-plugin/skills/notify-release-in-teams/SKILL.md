## 概要

カレントブランチと `production` ブランチの差分を調査し、社内利用ユーザー向けのTeamsリリース通知文をプレーンテキストで出力するスキル。

## 手順

### 1. 差分の取得（diff-analyzer サブエージェント）

`diff-analyzer` サブエージェントを呼び出して、productionブランチとの差分を機能単位で構造化した形で取得する。差分データはメインコンテキストを圧迫するため、サブエージェントに隔離して解析させる。

```
Agent ツールで subagent_type: "diff-analyzer" を起動する:
  prompt: "カレントブランチと production の差分を機能単位で解析し、
           変更内容のサマリーと機能別変更内容を返してください。"
```

サブエージェントの出力（差分サマリー・機能別変更内容）を次のステップのインプットとして使用する。

### 2. 変更内容を社内ユーザー向けに要約する

スクリプトの出力をもとに、変更内容を**社内の一般利用ユーザー**が理解できる言葉で短文にまとめる。

- 技術用語・実装の詳細は使わない
- 「何が変わったか」「どう便利になったか」の観点で書く
- 1項目につき1〜2文の短文にする
- 例（悪い）: 「APIのレスポンスキャッシュ機構をRedisに移行」
- 例（良い）: 「画面の読み込み速度が改善されました」

### 3. テンプレートを埋めて出力する

以下のテンプレートに沿い、開発内容の箇条書きを埋めて出力する。

テンプレート構造：

```
表題の件、リリースいたします。

リリースの内容
1. （変更点1）
2. （変更点2）

よろしくお願いいたします🙇‍♀️
```

### 4. 出力

マークダウン記法は一切使わず、プレーンテキストとしてそのまま出力する。
コードブロックや `#` などの記号は不要。
