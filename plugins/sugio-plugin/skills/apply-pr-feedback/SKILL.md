# PRフィードバック適用スキル

引数 `$ARGUMENTS` から以下を読み取る:

- **第1引数**: サービス名（`github` または `bitbucket`）
- **第2引数**: PR ID（番号またはURL）

---

## ステップ1: PRレビューデータの取得（pr-fetcher サブエージェント）

`pr-fetcher` サブエージェントを呼び出してPRデータを取得する。PRの差分＋コメントは数千〜数万行になることがあるため、メインコンテキストから隔離するためにサブエージェントを使用する。

```
Agent ツールで subagent_type: "pr-fetcher" を起動する:
  prompt: "以下のPRのデータを取得してください。
           サービス: {github または bitbucket}
           PR ID: {PR番号またはURL}

           PR基本情報・レビューコメント（インライン含む）・PR差分を
           構造化した形で返してください。"
```

サブエージェントの出力（PR基本情報・コメント一覧・差分）を次のステップのインプットとして使用する。エラーが返された場合はユーザーに提示して対処法を案内する。

---

## ステップ2: レビュー内容の解説

取得したデータをもとに以下の構成で解説する:

### 2-1. 全体サマリー

- レビュアーが何を懸念しているか（1〜3文）
- 承認状態（Approved / Changes requested / Comment only）

### 2-2. 指摘事項の詳細解説（指摘ごとに）

| 項目         | 内容                                           |
| ------------ | ---------------------------------------------- |
| 場所         | ファイルパス・行番号                           |
| 指摘内容     | コメントの要旨                                 |
| 技術的背景   | なぜそれが問題か                               |
| 修正の方向性 | どう直すか（複数ある場合はトレードオフも記載） |

### 2-3. 優先度分類

- **必須対応**: マージブロッカー・バグ・セキュリティ
- **強く推奨**: 設計・保守性の改善
- **任意対応**: スタイル・微細なニュアンス

---

## ステップ3: 修正案の並列生成（fix-proposal-agent × 3）

指摘事項の解説が完了したら、`fix-proposal-agent` サブエージェントを **3体同時に** 起動する。それぞれ異なるアプローチを担当し、独立した修正案を返す。並列実行により待ち時間を最小化できる。

Agent ツールで `subagent_type: "fix-proposal-agent"` を3つ同時に起動する:

**Agent A（保守的）:**
```
Agent ツールで subagent_type: "fix-proposal-agent" を起動する:
  prompt: "アプローチ: 保守的

           以下のPR差分とレビューコメントに基づいて修正案を作成してください。
           最小変更・既存コードを最大限保持する方針で対応してください。

           ## PR差分
           {ステップ1で取得した差分}

           ## レビューコメント（指摘事項）
           {ステップ2で整理した指摘リスト}"
```

**Agent B（積極的）:**
```
Agent ツールで subagent_type: "fix-proposal-agent" を起動する:
  prompt: "アプローチ: 積極的

           以下のPR差分とレビューコメントに基づいて修正案を作成してください。
           リファクタリングや設計改善を含めた対応をしてください。

           ## PR差分
           {ステップ1で取得した差分}

           ## レビューコメント（指摘事項）
           {ステップ2で整理した指摘リスト}"
```

**Agent C（バランス）:**
```
Agent ツールで subagent_type: "fix-proposal-agent" を起動する:
  prompt: "アプローチ: バランス

           以下のPR差分とレビューコメントに基づいて修正案を作成してください。
           変更量は中程度で、テスタビリティと保守性を重視してください。

           ## PR差分
           {ステップ1で取得した差分}

           ## レビューコメント（指摘事項）
           {ステップ2で整理した指摘リスト}"
```

---

## ステップ4: 修正案の比較と推奨

各エージェントの結果を受け取ったら以下を行う:

### 比較表の作成

| 観点                   | Agent A（保守的） | Agent B（積極的） | Agent C（バランス） |
| ---------------------- | ----------------- | ----------------- | ------------------- |
| 変更量                 | 少                | 多                | 中                  |
| リスク                 | 低                | 高                | 中                  |
| 保守性改善度           | 低                | 高                | 中                  |
| レビュー指摘への対応度 | ...               | ...               | ...                 |

### 推奨案の提示

- **推奨**: Agent [X] のアプローチ
- **理由**: （コードベースの状況・リスク・締め切りなどを加味した根拠）
- **採用時の注意点**: 追加で確認すべきこと

### オプション: ハイブリッド案

A・B・Cの良い部分を組み合わせた最終案があれば提示する。

---

## 注意事項

- **コードの変更はしない**: このスキルは提案フェーズ。実装はユーザーの承認後。
- **環境変数が不足している場合**: 取得できなかった旨を明示し、手動での情報提供を依頼。
- **大量のコメントがある場合**: 優先度「必須対応」のものに絞ってサブエージェントに渡す。
