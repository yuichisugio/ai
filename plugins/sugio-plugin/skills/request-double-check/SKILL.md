# ダブルチェック依頼

変更後の内容について、別観点での再確認が必要なときに使う。

## 引数

- 第1引数: 再確認したい観点。例: `UI`, `権限`, `リリース影響`, `例外処理`, `テスト不足`

## やること

1. 現在の変更内容を `git diff --stat` や対象ファイル確認で短く要約する。
2. 指定された観点、または差分から重要そうな観点を 1 つか 2 つに絞る。
3. その観点で再レビューし、バグ、回帰、見落とし、追加テストの要否を洗い出す。
4. 問題がなければ「大きな懸念なし」と明示し、残るリスクだけを簡潔に伝える。

## 守ること

- ユーザーが求めていない限り、ここでは編集やコミットは行わない。
- 指摘は根拠つきで伝える。推測なら推測だと明示する。
- 重要度順に結果を返す。
