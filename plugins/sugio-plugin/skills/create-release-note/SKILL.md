## 概要

カレントブランチと `production` ブランチの差分を調査して、リリースノートテンプレートを埋めてMarkdownで出力するスキル。

## 手順

### 1. 差分の取得（diff-analyzer サブエージェント）

`diff-analyzer` サブエージェントを呼び出して、productionブランチとの差分を機能単位で構造化した形で取得する。差分データはメインコンテキストを圧迫するため、サブエージェントに隔離して解析させる。

```
Agent ツールで subagent_type: "diff-analyzer" を起動する:
  prompt: "カレントブランチと production の差分を機能単位で解析し、
           変更内容のサマリー・機能別変更内容・ユーザーへの影響を返してください。
           リリースノート作成に使用します。"
```

サブエージェントの出力（差分サマリー・機能別変更内容・ユーザー影響）を次のステップのインプットとして使用する。

### 2. 変更内容を要約する

スクリプトの出力をもとに、変更の本質を2行程度で簡潔にまとめる。

- 技術的な詳細より「何を・なぜ変えたか」の観点でまとめる
- ユーザーや運用担当者が読んでわかる言葉を使う
- 例: 「ユーザー認証フローを刷新し、JWT方式に移行」「売上レポート画面のパフォーマンスを改善」

### 3. テンプレートを埋めてMarkdownで出力する

`skills/create-release-note/template.md` のテンプレートを使い、以下の項目を埋めて出力する。

| 項目                           | 埋め方                                              |
| ------------------------------ | --------------------------------------------------- |
| `YYYY.MM.DD タイトル名`        | 今日の日付 + ブランチ名や変更内容から適切なタイトル |
| `## ブランチ` のコードブロック | カレントブランチ名                                  |
| `開発内容`                     | 手順2で作成した2行程度の要約                        |
| `開発担当` / `リリース監督役`  | 不明な場合は `TBD` と記載                           |

**`## リリース作業` 以降のセクションは、特別な指示がない限り出力しない。**
ユーザーから「リリース作業も含めて」などの指示があった場合のみ出力する。

### 4. 出力

Claude Codeのコンソールはマークダウンをレンダリングするため、`#` や ` ``` ` などの記号がそのまま表示されない。
コピペで使えるリリースノートにするため、**完成したリリースノート全体を1つのコードブロック（` ```markdown `）で囲んで出力する**。

出力フォーマット（このまま厳守）：

````
```markdown
# 2026.03.24 タイトル名

## ブランチ

```
feature/xxx
```

## リリース内容

- 開発内容
  - XXX
  - YYY
- 開発担当
  - 杉尾
- リリース監督役
  - TBD
```
````
