# authテスト環境デプロイスキル

引数 `$ARGUMENTS` から以下を読み取る:

- **第1引数**: デプロイ対象（`front` または `back`）

引数が不正・未指定の場合は、ユーザーに `front` または `back` のどちらかを指定するよう案内して終了する。

---

## ステップ1: 現在の状態を確認・表示する

以下のコマンドで現在の状態を取得する:

```bash
git branch --show-current
git remote get-url origin
```

取得した情報をもとに、以下の内容を**ユーザーが目視で確認できるよう明示的に表示**する:

| 項目                             | 値                                         |
| -------------------------------- | ------------------------------------------ |
| デプロイ対象                     | `front` または `back`                      |
| リポジトリURL                    | `git remote get-url origin` の出力         |
| 現在のブランチ                   | `git branch --show-current` の出力         |
| 作成されるテストブランチ（予定） | `test/YYYY-MM-DD-{ブランチ名サフィックス}` |

---

## ステップ2: ユーザーに確認を求める

> **VSCodeワークスペースで複数リポジトリを開いている場合、誤ったリポジトリへのデプロイを防ぐために、必ず確認を取る。**

以下のメッセージでユーザーに確認を求める（**必ずユーザーの応答を待ってから次に進む**）:

```
上記のリポジトリ・ブランチにデプロイします。
VSCodeワークスペースで複数のリポジトリを開いている場合は、
対象リポジトリが正しいことを確認してください。

このまま続行しますか？ [y/N]
```

- ユーザーが `y` または `yes`（大文字小文字不問）を入力した場合のみ次のステップへ進む
- それ以外（`n`、`no`、その他）の場合はデプロイを中断し、「デプロイをキャンセルしました」と伝えて終了する

---

## ステップ3: デプロイスクリプトを実行する

スキル読み込み時に示された `Base directory for this skill:` のパスを `SKILL_BASE_DIR` として、以下を実行する:

```bash
bash "${SKILL_BASE_DIR}/scripts/deploy-test-env.sh" "$TARGET"
```

**スクリプトが行うこと:**

1. 引数（`front` / `back`）の検証
2. デプロイ情報（対象・リポジトリ名・ブランチ）の表示
3. `production` ブランチをチェックアウトして最新化
4. `test/YYYY-MM-DD-{サフィックス}` ブランチを作成してカレントブランチをマージ
5. テストブランチをリモートへプッシュ

スクリプトがエラーを返した場合は、エラーメッセージをユーザーに提示し、考えられる原因（コンフリクト・認証エラー等）と対処法を案内する。

---

## ステップ4: 結果を報告する

デプロイ成功時は以下の情報をまとめて報告する:

- プッシュしたテストブランチ名
- デプロイ対象（front / back）
- リポジトリURL

---

## 注意事項

- **必ずステップ2の確認を取ること**: 確認なしにデプロイスクリプトを実行しない
- `production` ブランチへの直接変更は行わない（スクリプトはテストブランチを新規作成する）
- マージコンフリクトが発生した場合は、ユーザーに手動解消を依頼する
